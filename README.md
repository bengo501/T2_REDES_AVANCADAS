# trabalho de análise de vulnerabilidades

trabalho prático da disciplina de redes de computadores avançadas - pucrs.

## objetivo

realizar e documentar análise de vulnerabilidades em dois alvos reais, identificando falhas de segurança em redes, sistemas ou aplicações.

## estrutura do projeto

- `enunciado.txt` - enunciado do trabalho
- `relatorio.md` - template do relatório a ser preenchido
- `guia_ferramentas.md` - guia de ferramentas e técnicas utilizadas
- `README.md` - este arquivo

## como usar

1. **escolher os alvos:**
   - selecionar dois equipamentos da rede doméstica ou do laboratório (sala 301)
   - obter autorização para realizar a análise

2. **realizar a análise:**
   - seguir as etapas descritas no relatório
   - consultar o guia de ferramentas para saber quais ferramentas utilizar
   - documentar todos os passos e resultados

3. **preencher o relatório:**
   - usar o template `relatorio.md`
   - preencher com os resultados obtidos
   - adicionar screenshots e evidências nos anexos

4. **revisar e entregar:**
   - revisar o relatório completo
   - garantir que todos os pontos do enunciado foram atendidos
   - preparar apresentação (data: 11/11)

## etapas do trabalho

### etapa 1: obtenção de informações
- usar ferramentas públicas e técnicas de varredura
- coletar informações básicas dos alvos (whois, dns, etc.)

### etapa 2: mapeamento da rede e identificação de serviços
- identificar serviços expostos
- detectar possíveis vulnerabilidades
- mapear superfície de ataque

### etapa 3: documentação, análise de impacto e propostas
- analisar impacto de cada vulnerabilidade
- propor correções e mitigações
- documentar tudo no relatório

## requisitos

- grupo de até 4 componentes
- todos devem estar presentes na apresentação
- relatório completo conforme estrutura especificada

## ferramentas recomendadas

consulte `guia_ferramentas.md` para lista detalhada de ferramentas e como usá-las.

ferramentas principais:
- nmap (varredura de portas)
- nikto (vulnerabilidades web)
- metasploit (framework de testes)
- wireshark (análise de tráfego)

## aviso importante

⚠️ **sempre obter autorização** antes de realizar qualquer varredura ou teste em sistemas. este trabalho é apenas para fins educacionais em ambientes controlados e autorizados.
